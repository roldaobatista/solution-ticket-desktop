## 5. SEO técnico

### Schema.org

- **Organization** (raiz solution-ticket.com): logo, sameAs (LinkedIn, YouTube), contato.
- **Product** (páginas de pricing/módulo): nome, descrição, ofertas (faixas), reviews quando existirem.
- **Article** (cada post): headline, datePublished, author, image, mainEntityOfPage.
- **FAQPage** em pillar pages.
- **VideoObject** em páginas com webinar.

### Sitemap & robots

- `sitemap.xml` autogerado, atualizado a cada deploy.
- `robots.txt` permitindo crawl total exceto `/internal/`, `/admin/`, `/preview/`.
- `<link rel="canonical">` em todas as páginas.

### Performance (Core Web Vitals)

| Métrica                         | Alvo    | Limite aceitável |
| ------------------------------- | ------- | ---------------- |
| LCP (Largest Contentful Paint)  | ≤ 2,0s  | ≤ 2,5s           |
| CLS (Cumulative Layout Shift)   | ≤ 0,05  | ≤ 0,10           |
| INP (Interaction to Next Paint) | ≤ 150ms | ≤ 200ms          |
| TTFB (Time to First Byte)       | ≤ 500ms | ≤ 800ms          |

**Stack sugerida site marketing**: Next.js 14 estático (SSG/ISR) em Cloudflare Pages, imagens em AVIF/WebP, fontes auto-hospedadas com `font-display: swap`.

### On-page

- Title tag ≤ 60 caracteres, com keyword principal nos primeiros 50.
- Meta description ≤ 155 caracteres, com CTA.
- H1 único por página, contendo keyword principal.
- URLs em kebab-case, sem stopwords desnecessárias (`/blog/integracao-balanca-erp` não `/blog/como-fazer-a-integracao-de-balanca-com-erp`).
- Alt text em todas as imagens.
- Internal linking obrigatório: cada artigo linka 2+ artigos do mesmo cluster e 1 pillar.
